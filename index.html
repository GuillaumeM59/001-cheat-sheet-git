Premiers pas sur GIT

  • Remplacez toutes les * par git !
  • * init initialise git dans ton nouveau projet.
  • * status permet de voir les modifs non suivies et celles qui le sont deja.
  • * add 
    #fichier
    ou 
    --all
     ajoute le 
    #fichier
    au suivi, 
    --all
    les ajoute tous.

Let's commit!

  • Remplacez toutes les * par git !
  • * commit -m "
    message"
     enregistre toutes les modifications (en vert dans statut) et lui associe un numero unique et un message de description.
  • * log voir l'historique des commits.
  • * diff voir les changements depuis le dernier commit.
  • * checkout 
    #
     revenir visuellement au commit 
    #
    .
  • * checkout master revenir au dernier commit ( apres checkout 
    #
    ).
  • * checkout 
    #fichier
     revenir comme avant.
  • * revert 
    #
     annule le commit 
    #
    .
  • * reset retour definit d'un commit en arrière.
  • * reset HEAD ^^^... retour definitif en arrière d'autant de commit que de ^.

Partageons!

  • Remplacez toutes les * par git !
  • * push 
    #hôte
    #branche
     envoie la dernière version à l'hôte 
    #hôte
    sur la 
    #branche
    .
  • * pull 
    #destination
    #branche
     recupère la dernière version et la met sur votre machine.
  • * clone 
    #adresse
     crée une copie du projet 
    #adresse
    .
  • * blame

Faisons une pause...

  • Remplacez toutes les * par git !
  • * stash met les dernière modifications en memoire tampon permettant de faire un commit n'ayant aucun rapport.
  • * stash pop revient exactement comme au moment du stash (en conservant les commit réalisé entre).
